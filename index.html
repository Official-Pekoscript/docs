Quick Start

Start by downloading the installer

Pekoscript doesn't have an installer, must be built from source.

Create a file named main.peko and add the contents

func main() {
    std::io::print("Hello world");
}

Then compile and run with with:

psc main.peko
./a.out # for unix
.\a.exe # for windows

Pekoscript

Simple Expressions

For mathematical expressions, pekoscript supports the +, -, *, /, (), and % operators. Pekoscript also follows PEMDAS order.

Pekoscript also offers the ==, !=, >=, <=,>, and, or, and < operators for boolean expressions.

Built-In Types:

  • str -> For string literals
  • char -> For character literals
  • num -> For integer and float literals
  • size -> For unsigned integer literals (integers that can't be less than 0)
  • bool -> For boolean values
  • void -> Emtpy type
The following section is not needed to use pekoscript generally.

Built-In types for C ffi:

  • cint -> Equivalent to a normal 32-bit integer
  • cvoid -> Void pointer type
  • size -> Equivalent to a long in C

Pekoscript also has true and false keywords for boolean values


Variables

Variables can be global or local, although some values will not work for global variables.

To declare a variable, the syntax is:

let variable_name: variable_type = value;

If you want to declare a variable with type inference:

let variable_name = value;

If you want to declare multiple variables with the same value:

let variable_name, variable_name2, ...: variable_type = value;

If you want to declare a variable with no value:

let variable_name: variable_type;
In order to declare a variable with no value, a type must be provided

To set a variables value after declaration, use the syntax:

variable_name = value;

Functions

Functions are declared with arguments and a type. If no type is provided, the function will be seen as void (no return).

func function_name(arg1: type): type {
    ...
}

To forward-declare a function, replace the body with a semicolon:

func function_name(arg1: type): type;

adding the ext keyword after the func creates the function as an extern (or defined in a seperate file). This also doesn't add name-mangling to the function


Control Flow

Pekoscript has if statements

if condition {
    ...
}

If you want to create an else-if branch, use the elif keyword

if condition {
    ...
} elif condition {
    ...
}

If you want to create an else branch, use the else keyword

if condition {
    ...
} else {
    ...
}

For looping, Pekoscript currently only supports while loops. They will loop until the provided condition is false

while condition {
    ...
}

To break out of a loop before the condition is false, use the break keyword

while condition {
    ...
    break;
}

To return from a function, use the return keyword

func function_name(...): ... {
    ...
    return;
}

Classes

Declaring a class

Classes in pekoscript are used to store data and call methods on that data.

The default constructor for a class in pekoscript can be created by defining a function with the name __init__

To create a class, the syntax is:

class class_name {
    data1: type;
    
    func __init__() {
        ...
    }

    func func1(): ... {
        ...
    }
}

You can add as many data/attributes and methods to the class as you want.

To access the classes attributes from its methods, use the this keyword

Example:

class Person {
    name: str;

    func __init__(name: str) {
        this.name = name;
    }

    func sayHi() {
        std::io::print("Hello, my name is ");
        std::io::println(this.name);
    }
}
Inheritance
Only single inheritance is supported in Pekoscript currently

The syntax to inherit from a class is as follows:

class derived : base {
    ...
}

When a class is inherited, the derived class gets all of the base classes attributes. Additionally, the derived class inherits the base classes methods unless overidden in the derived class.

Example:

class Person {
    name: str;

    func __init__(name: str) {
        this.name = name;
    }

    func sayHi() {
        std::io::print("Hello, my name is ");
        std::io::println(this.name);
    }
}

class Worker : Person {
    job: str,

    func sayHi() {
        std::io::print("Hello, I am a worker and my name is ");
        std::io::println(this.name);
    }

    func sayJob(job: str) {
        this.job = job;

        std::io::print("Hello, my job is ");
        std::io::println(job);
    }
}
To create an instance of a class, call the class name as a function:
let instance = class_name(constructor_args)

Pekoscript also offers operator overloading.

The list of supported overloads are:

  • func __brace__(index): type
    ↳ overload array access for operator []
  • func __brace_set__(index): &type
    ↳ overloads array setting for operator []=. This overload should return a reference to the value at the index. This can be done with the & before the value.
  • func __add__(rhs): type
    ↳ overload for operator +, should return a value.
  • func __sub__(rhs): type
    ↳ overload for operator -, should return a value.
  • func __mul__(rhs): type
    ↳ overload for operator *, should return a value.
  • func __div__(rhs): type
    ↳ overload for operator /, should return a value.
  • func __eq__(rhs: num): type;
    ↳ overload for ==
  • func __neq__(rhs: num): type
    ↳ overload for !=
  • func __gt__(rhs: num): type
    ↳ overload for >
  • func __lt__(rhs: num): type
    ↳ overload for <
  • func __geq__(rhs: num): type
    ↳ overload for >=
  • func __leq__(rhs: num): type
    ↳ overload for <=
  • func __not__(rhs: num): type
    ↳ overload for !
  • func __and__(rhs: num): type
    ↳ overload for and
  • func __or__(rhs: num): type
    ↳ overload for or
  • func __to_string__(rhs: num): str
    ↳ overload for boxing the object as a str
  • func __to_num__(rhs: num): num
    ↳ overload for boxing the object as a num
  • func __to_size__(rhs: num): size
    ↳ overload for boxing the object to a size
  • func __to_bool__(rhs: num): bool
    ↳ overload for boxing the object to a bool

Generics

Pekoscript has support for generics. Generics can be defined as classes, although two differences is that they take type parameters and cannot inherit.

To create a generic, the syntax is almost the same as a class declaration:

class class_name<typeparam1, typeparam2, ...> {
    data1: typeparam1;
    data1: typeparam2;
    
    func __init__() {
        ...
    }
}

To create an instane of a generic class, it is very similiar to class instance creations:

let instance = class_name: <type>(constructor_args)

Modules

There are two ways to create modules in pekoscript: 1) By defining it with special syntax, or 2) Create a file and import it as a module

To use the built-in syntax, use the module keyword and define methods and variables inside module body

module module_name {
    ...
}

To access the contents of a module, use the syntax:

module_name::method() or module_name::variable

To import files, use the import keyword

The import keyword will search for files in the current directory first, and then search the installed packages after.

To import a module, use the syntax

import filename;

To import a module from a folder:

import folder.filename;
Pekoscript does not currently support importing entire folders

To import a module from a specific path:

import "specificpath";

To import a module as a certain name, use the syntax:

import filename as modulename;
Modules cannot be imported as "object" because that is a special keyword for linking

Platform Expressions

Platform expression are simply expressions that only get evaluated if the provided platform name is equal to the current compilation target.

platform "platformname" {
    ...
}

The platform name can either be "osx", "win32", or "linux"

Anything can be defined in the platform expressions body

Standard Library

String class

The string class is a wrapper aroudn the str datatype

To instantiate a String, 

std::String("value")

The class offers the following methods:

  • std::String.push(str)
    ↳ appends the provided string to the end
  • std::String.push_front(str)
    ↳ appends the provided string to the front
  • std::String.push_char(char)
    ↳ appends the provided char to the end
  • std::String.len(): size
    ↳ returns the length of the string
  • std::String.to_num(): num
    ↳ returns the string as a num
  • std::String.to_size(): size
    ↳ returns the string as a size

Array class

The array class is a generic that can store a collection of the provided type

To instantiate an Array, 

std::Array: <type>()

The array class also uses overloads so that you can access and set like an array:

let array = std::Array: <str>()
array.push("hello world")
array[0] = "Hello World";
std::io::print(array[0]);

The class offers the following methods:

  • std::Array.push(type)
    ↳ appends the provided type to the end
  • std::Array.allocate(size)
    ↳ allocates an array of the specified size
  • std::Array.push_alloc()
    ↳ allocates on space at the end of the array
  • std::Array.remove(size)
    ↳ removes the element from the index
  • std::Array.set(size)
    ↳ sets the element at the index
  • std::Array.get_elem_index(T)
    ↳ finds the index of the given element or returns -1

Map class

The map class is a generic that can be indexed by a given type to a piece of data

To instantiate a Map, 

std::Map: <type1, type2>()

The Map class also uses overloads so that you can access and set like an array:

let map = std::Map: <str, num>()
map.insert("item1", 10);
map["item1"] = 20;
std::io::println(map["item1"]);

The class offers the following methods:

  • std::Map.insert(KT, VT)
    ↳ inserts the provided value at the index
  • std::Map.contains(KT): bool
    ↳ returns true if the key is in the map, returns false otherwise

Pair class

The Pair class is a generic that holds two pieces of data of the provided types

To instantiate a Pair, 

std::Pair: <type1, type2>(data1, data2)

To access the data pieces:

let pair = std::Pair: <str, num>("dude", 50)
std::io::println(dude.first); // prints dude
std::io::println(dude.second); // prints 14

IO module

The IO module provides 4 methods for handling input and output

  • std::io::fmt(std::String, std::Array<std::String>): str
    ↳ formats {} with the provided string
  • std::io::print(str)
    ↳ prints the provided string without a newline
  • std::io::println(str)
    ↳ prints the provided string with a newline
  • std::io::input(str): std::String
    ↳ prints the provided string and returns the input from the terminal as a str

CLI

The pekoscript CLI can be invoked by calling psc in your command line

Flags

The Pekoscript Compiler, which will now be reffered to as psc, has a few flags to help you with compiling:

  • -emit-ir
    ↳ emits the created ir code
  • -arch
    ↳ sets the target architecture. Can be x86_64, arm, or i686.
  • -os
    ↳ sets the target operating system. Can be x86_64, arm, or i686.
  • -preset
    ↳ sets the target operating system and architecture. Can be osx-m-chip, osx-i-chip, win32, lin-arm, or lin-x86_64.
  • -v
    ↳ Sets verbosity on.
  • -config
    ↳ Must be ran on install. Downloads required files for psc to run.
  • -update
    ↳ Updates the required files for pekoscript to run.
  • -clang-flags
    ↳ when used in conjuction with 1) the -arch and -os flags, or 2) the -preset flag psc will output the flags needed for clang to output the proper object code file for the platform. This will also be covered at cross compiling

Cross Compiling

Pekoscript is versatile and currently supports compiling to Windows (x86_64 and arm), MacOS (both Intel and M-Chip series), and Linux (both x86_64 and arm). provides a few options for compiling.

To set your target os for:

  • Windows: pass win32 to the -os flag
  • MacOS: pass osx to the -os flag
  • Linux: pass linux to the -os flag

To set your target architecture for:

  • Arm: pass arm to the -arch flag
  • i686 (for windows): pass i686 to the -arch flag
  • x86_64: pass x86_64 to the -arch flag

Some presets for:

  • MacOS with an M-Series chip: osx-m-chip
  • MacOS with an intel chip: osx-i-chip
  • Linux with an arm chip: lin-arm
  • Linux with an x86_64 chip: lin-x86_64
  • Windows: win32

Linking with other Languages

Pekoscript offers a few tools to help with linking with other languages.

To demonstrate this, we link a C file with Pekoscript

First, create your C code:

// file: print.c
#include <stdio.h>

void print(char* to_print) {
    printf("%s\n", to_print);
}

To compile this, use psc with the preset or arch+os flags in conjuction with the -clang-flags flag to get the proper flags.

clang -c print.c -o print.o `psc -arch arch -os os -clang-flags`

Next, we will create the pekoscript binding for this:

// file: print.peko
import print as object

func ext print(to_print: str);

func main() {
    print("Hello World!");
}

You will notice something new, which is the import ... as object. when you import a file as an object, psc will search for the .o file and link it automatically.

Compile the code with psc and run it:

psc -arch arch -os os print.peko
./a.out
Hello World!

You have successfully linked your first outside program with pekoscript.

PekoUI

Importing PekoUI

To import the PekoUI module, write:

import pekoui as ui

XML Syntax

Pekoscript supports a very simple XML Syntax currently

It starts with the PekoX flag

<PekoX></PekoX>

Any valid html tag can be used between the PekoX tag.

It is possible to insert values by using braces, for example:

let name = "Dude";
<PekoX>
    <h1>Hello, my name is {name}</h1>
</PekoX>

Any valid Pekoscript expression can be put in the braces.

Creating a Page

To create a page in Pekoscript, you can create a class that inherits from pekoui::PekoPage:

class MainPage : ui::PekoPage {
    // define attributes

    // define methods
}

PekoPage has a few methods that can be used:

  • func render(ui::PekoApp): str
    ↳ returns the final rendered html
  • func style(ui::PekoApp): str
    ↳ returns the final css styling

Adding Callbacks

Callbacks are created in the xml syntax, but can only be created in a PekoPage object.

To add callbacks, first create the method in the class.

Callbacks take a list of arguments from the javascript, and can return a string with a JSON object.

After, add it in the xml as an attribute:

class MainPage : PekoPage {
    ...
    func callback(args: std::Array<std::String>, peko_app: cvoid): str {
        let app: ui::PekoApp = ui::get_app(peko_app);
        ...
    }

    func render(app: ui::PekoApp) {
        return <PekoX>
            <button onclick="{callback}">Click Me</button>
        </PekoX>
    }
}
The render method should always have an argument with the exact name "app" that has the type of a ui::PekoApp. Also, callbacks take an argument of cvoid that contains the reference to the page.

Creating the App

The pekoui library has a class named PekoApp. It is what will be used to instantiate and hold pages.

Lets create an example pekoscript app for this:

Start off by creating your page. We'll create a counter app:

class MainPage : PekoPage {
    counter: num,
    
    func __init__() {
        this.counter = 0;
    }

    func increment(args: std::Array<std::String>, peko_app: cvoid): str {
        let app = get_app(peko_app);
        this.counter += 1
        app.reload(); // update the ui to show the new counter
        return "";
    }

    func render(app: ui::PekoApp) {
        return <PekoX>
            <h1>Counter is {this.counter}</h1>
            <button onclick="{increment}">Add to counter</button>
        </PekoX>
    }
}

Next, we'll create the app in the main function:

func main() {
    let app = ui::PekoApp("First Peko GUI", 800, 800) // the two numbers are the width and height of the window
    app.add_page("index", MainPage()) // insert the page as the index, the first page to be rendered
    app.start() // start the app
}

Compile this psc, run it, and you will have your first Web based UI running
